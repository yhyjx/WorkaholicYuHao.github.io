# 基本内容

# 模版语法

Vue3 入门的基本语法。主要是用来将组件上的实例数据与 DOM 元素相关联，最终被游览器解析展示在页面上。

# Mustache 语法

双大括号语法是用来将组件实例的数据绑定到需要展示的 DOM 上，每次更改该数据会同步更新。

# v-html

双大括号语法会将 html 插值为文本而不是 html，当我们想要插入元素的时候需要使用 v-html 指令。

<h1>{{rawHtml}}</h1>
<div v-html="rawHtml"></div>
<script>
data(){
	return {
		rawHtml: `<span>我是动态 html 。</span>`
	}
}
</script>

# v-bind

用来响应式绑定 html 上的 attributes 属性。当属性绑定的数据为 null / undefined 该属性不显示。

<div v-bind:class="myClass">全写绑定属性 class</div>
<div :class="myClass">简写绑定 class</div>
<div v-bind="objAttrs">同时绑定多个元素</div>
<div :[name]="url">动态选择属性</div>
<button @[clickName]="doSomeThing"></button>
<span :style="{'color': red,'font-weight': 600}">红色 600</span>
<span :style="[{color: green},{'font-weight': 800}]">绿色 800</span>
<script>
  data(){
    return {
      myClass: 'commonClass',
      objAttrs: {
        id: 'myId',
        class: "myClass"
      },
      name: "href",
      clickName: 'click'
    }
  }
</script>

# 响应式基础

修改数据从而达到同步修改界面显示。

# 数据绑定

data 选项用来声明组件的响应式状态，该值为返回一个对象的函数，在创建新的组件实例时调用此函数，函数内部的属性被代理到了组件实例上。

export default {
  data(){
    return {
    	// 写入需要绑定的属性与值。
    }
  }
}

# 声明方法

methods 选项，用来包裹所有方法的一个对象。Vue 会默认将对象中的方法只想组件实例。

# 计算属性

当我们有多复用表达式的时候，可以使用计算属性，这样代码不会过于臃肿难以维护，计算属性是具备缓存的只会在发生改变的时候执行一次。

<div v-text="computedTxt"></div>
<div v-text="computedTxt"></div>
<div v-text="computedTxt"></div>
export default {
  computed: {
    computedTxt(){
      return '只会执行一次'
    }
  }
}

# 可劫持计算属性

计算属性是可以支持通过设置 getter 和 setter 控制读取和写入的。

export default {
  computed: {
    data(){
      return{
        firstTxt: '可以',
        lastTxt: '控制读取'
      }
    },
    computed: {
      computedTxt: {
        get(){
        	return this.firstTxt + this.lastTxt
      	},
      	set(newVal){
        	// 通过 , 分割新设置的值来分别赋值。
        	[this.firstTxt,this.lastTxt] = newVal.split(',')
      	}
      }
    }
  }
}

# 条件渲染

# v-if

用于条件性的渲染一块内容，当为 true 的时候才会渲染。

# v-show

另一个用来条件渲染一块内容的指令。

# v-if 和 v-show 的区别

v-if 只有条件为真的时候才会渲染到界面上。

v-show 始终会被渲染到界面上，只是简单的切换元素 display 的显隐。

# 列表渲染

# v-for

当有数组或对象格式的数据出现时，可以使用 v-for 渲染列表。

# v-if 和 v-for

不推荐同级使用 v-if 和 v-for ,当二者同时存在的时候，前者要比后者的优先级更高，他无法访问 v-for 中的别名。

# 通过 key 管理状态

默认情况下列表的更新高效的，但只适用于列表渲染不依赖于子组件状态。加上 key 会保证时刻高效。

# 事件处理

# 事件修饰符

修饰符是以点开头的特殊后缀，表明指令需要一些特殊的方式绑定

  • .prevent: 会对触发事件调用 event.preventDefault()。
  • .stop: 事件停止传递。
  • .once: 只执行一次。
  • .self: 只有点击当前元素的时候才会触发。
  • .capture: 当事件触发时首先处理包裹事件在处理当前元素事件，相当于捕获。

# 按键修饰符

  • .enter: 点击回车触发。
  • .tab: 点击 tab 触发。
  • .delete: 点击删除键触发。
  • .space: 空格触发。
  • .up: 点击上键触发。
  • .down: 点击下键触发。

# 数据双向绑定

# v-model

变量与值之间相互绑定。多用于文本、输入框、下拉框等。

# 修饰符

  • .lazy: 默认情况下，v-model 会在 input 事件后更新数据，添加该修饰符需要等到 change 后才会更新数据。
  • .number: 输入自动转换为数字，type="number" 情况下自动添加。
  • .trim: 默认去除输入的空格。

# 监听器

当修改数据的时候想要触发一些方法，这个时候就会用到监听器选项，每一次数据更新触发一次对应方法。

export default {
  data(){
    return {
      name: '基础改变',
      newObj: {
        name: '深层改变'
      }
    }
  },
  watch:{
    name(oldVal,newVal){
      console.log(oldVal,newVal)
	},
    "newObj.name"(oldVal,newVal){
      console.log(oldVal,newVal)
    }
  }
}

# 回调的触发时机

当修改了数组时，会同时触发 Vue 组件更新和监听器回调。默认情况下监听器回调总是在 Vue 组件更新之前。如果想在监听器内部访问更新后的 DOM ,通过 flush: 'post' 实现。

watch:{
  newObj: {
    handler(oldVal,newVal){
      console.log(oldVal,newVal)
    },
    immediate: true,
    flush: 'post'
  }
}

# this.$watch()

我们可以用组件实例的 this.$watch() 方法命令式创建一个监听器。该监听器可以提前被停止。

export default {
  created() {
    this.$watch('question', (newQuestion) => {
    })
  }
}

# 停止监听器

默认情况下已经声明的监听器，会在组件卸载后销毁。少数情况下我们可以调用 $watch() 返回的函数来停止监听。

const unwatch = this.$watch('name',()=>{})
