# 初识 Vue3

# 1. Vue3 带来的变化

  • 从源码角度
    • 源码通过 monorepo 的形式来管理源代码。
      • Mono: 单个
      • Repo: repository 仓库
      • 主要是将多项目的代码存储在一个 repository 上。
      • 这样做的目的是多个包相互独立，可以有自己的功能逻辑、单元测试等，同时在同一个仓库下方便管理。
      • 模块划分的更加清晰，可维护性、可扩展性更强。
    • 源码通过 TypeScript 编写
  • 从性能角度
    • 使用 Proxy 进行数据劫持，在 Vue2.0 中使用 Object.defineProperty 来进行数据劫持但是无法劫持对象添加或删除属性的操作，后续增加了许多特殊的 API 比如 $set / $delete 等。
    • 删除了一些没有必要的 API ，如 filter 、内联模版等。
    • Slot 编译优化、diff 算法优化。
  • 新的 API
    • 由 Options API 到 Composition API。
    • Hooks 函数增加代码的可复用性。在 Vue2.x 的时候，我们通过 mixins 在多个组件中共享逻辑，但是多个组件会存在命名冲突。

# 4. template 属性

在 createApp 方法中传入了一个对象，其中 template 表示 Vue 需要渲染的模版信息。首先清空我们需要挂载元素的 innerHTML , 然后替换为传入的内容。

# template 使用的两种方式

  • 使用 script ,标记类型为 x-template。
<script type="x-template" id="template1">
  <h1>Hello template1</h1>
</script>

<script src="https://unpkg.com/vue@3"></script>
<script>
  Vue.createApp({
    template: "#template1",
  }).mount("#app");
</script>

# 5. data 属性

在 Vue2.0 中，data 可以为函数也可以为对象，但是函数会有警告。

在 Vue3.0 中，data 只能为对象，否则报错。

data 中返回的对象会被 Vue 的响应式系统所劫持，所以 template 中可以通过 访问，从对象中获取属性，同理当对象中的属性修改 template 就会被更新。

# 6. methods 属性

methods 属性是一个对象，我们会在对象内部定义许多方法。

这些方法可以被绑定到 template 模版中。

在方法中，我们通过 this 来访问 data 返回的对象属性。

# 为什么不能使用箭头函数定义方法

因为箭头函数执行上级作用域，默认为 Window 。所以 this 访问不到组件实例 。

在 Vue 3.x 源码中，会对 methods 对象中的属性就行遍历，并通过 bind 绑定当前实例对象在进行赋值。
